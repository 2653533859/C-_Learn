## 🛠️ 编译与运行指南

### 基本编译命令

```bash
# 最简单的编译
gcc filename.c

# 指定输出文件名
gcc filename.c -o output_name

# 编译多个文件
gcc file1.c file2.c -o program

# 显示所有警告（推荐）
gcc filename.c -o output_name -Wall

# 使用 C11 标准
gcc filename.c -o output_name -std=c11

# 编译时保留调试信息（用于 GDB）
gcc filename.c -o output_name -g
```

### 常用编译选项

| 选项 | 说明 | 推荐使用场景 |
|------|------|-------------|
| `-o` | 指定输出文件名 | 所有情况 |
| `-Wall` | 显示所有警告 | 所有情况 |
| `-Wextra` | 显示额外警告 | 代码质量检查 |
| `-g` | 包含调试信息 | 使用 GDB 调试 |
| `-O2` | 优化级别 2 | 发布版本 |
| `-std=c11` | 使用 C11 标准 | 使用新特性 |
| `-lm` | 链接数学库 | 使用 math.h 函数 |

### 运行程序

```bash
# Linux/macOS
./program_name

# Windows (MSYS2/MinGW)
./program_name.exe
# 或
program_name.exe
```

### 调试技巧

#### 1. 使用 printf 调试

```c
int x = 10;
printf("DEBUG: x = %d\n", x);  // 输出变量值
```

#### 2. 使用 GDB 调试器

```bash
# 编译时加 -g 选项
gcc program.c -o program -g

# 启动 GDB
gdb ./program

# 常用 GDB 命令
(gdb) break main       # 在 main 函数设置断点
(gdb) run             # 运行程序
(gdb) next            # 单步执行（不进入函数）
(gdb) step            # 单步执行（进入函数）
(gdb) print variable  # 打印变量值
(gdb) continue        # 继续执行
(gdb) quit            # 退出 GDB
```

#### 3. 常见编译错误

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `undefined reference to 'main'` | 缺少 main 函数 | 添加 main 函数 |
| `implicit declaration of function` | 未声明函数 | 添加函数声明或包含头文件 |
| `expected ';' before '}'` | 缺少分号 | 在语句末尾添加分号 |
| `undeclared identifier` | 变量未声明 | 在使用前声明变量 |
| `incompatible types` | 类型不匹配 | 检查赋值和函数参数类型 |

## 💡 学习建议与常见问题

### 学习建议

1. **动手实践最重要**
   - 不要只看代码，一定要自己敲一遍
   - 每个知识点都编写小程序验证
   - 尝试修改示例程序，观察结果

2. **理解优于记忆**
   - 不要死记硬背语法，理解其原理
   - 通过画图（流程图、内存图）帮助理解
   - 思考"为什么"而不只是"是什么"

3. **循序渐进，不要跳跃**
   - 基础语法是后续学习的基石
   - 遇到困难不要气馁，反复练习
   - 完成所有练习题再进入下一模块

4. **养成良好习惯**
   - 代码缩进对齐（使用 4 空格或 1 Tab）
   - 变量命名有意义（`student_count` 而非 `sc`）
   - 添加必要的注释
   - 编译时使用 `-Wall` 检查警告

5. **寻求帮助**
   - 遇到问题先查阅 guide.md 和 solutions.md
   - 使用搜索引擎和 Stack Overflow
   - 加入学习社区交流讨论

### 常见问题 FAQ

**Q1: 我需要提前学习其他编程语言吗？**  
A: 不需要。本模块从零开始，适合完全没有编程经验的学习者。

**Q2: 学习 C 语言需要购买什么书籍吗？**  
A: 本模块的资料已经足够入门。如需深入，推荐《C Primer Plus》或《C 程序设计语言》（K&R）。

**Q3: 编译时出现 "undefined reference to 'sqrt'" 怎么办？**  
A: 数学函数需要链接数学库，使用 `gcc program.c -o program -lm`。

**Q4: 为什么我的程序运行结果和示例不一样？**  
A: 检查以下几点：
   - 是否正确输入了数据
   - 数据类型是否匹配（整数除法 vs 浮点除法）
   - 变量是否初始化
   - 运算符优先级是否正确

**Q5: 练习题太难了，完全没有思路怎么办？**  
A: 
   1. 先完成所有简单题（🟢）建立信心
   2. 阅读 guide.md 中的相关章节
   3. 在纸上画出解题思路
   4. 查看 solutions.md 中的提示（不要直接看完整答案）
   5. 参考示例程序中的类似代码

**Q6: 递归太难理解了！**  
A: 
   - 先掌握循环，理解递归需要时间
   - 画出递归调用树（如 solutions.md 中的示例）
   - 使用 GDB 单步跟踪递归过程
   - 多做几道递归题，培养递归思维
